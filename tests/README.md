# SoleStyle ShoeStore — Test Suite

Automated test suite for the SoleStyle e-commerce application, generated via [kane-cli](https://github.com/LambdaTest/kane-cli).

## Structure

```
tests/
├── README.md                  # This file
├── .env.example               # Shared environment config for Playwright scripts
├── requirements.txt           # Shared Python dependencies for Playwright scripts
├── test-cases/                # Kane-cli testmd source files (the test definitions)
│   ├── 01_homepage_load_test.md
│   ├── 02_product_search_test.md
│   ├── ...
│   └── 10_info_pages_test.md
├── generated-scripts/         # Playwright Python scripts generated by kane-cli
│   ├── test_tc01_homepage.py
│   ├── test_tc03_product_detail.py
│   ├── ...
│   └── test_tc10_info_pages.py
└── results/                   # Test run results (pass/fail per step)
    ├── tc01_homepage_result.md
    ├── tc02_search_result.md
    ├── ...
    └── tc10_info_pages_result.md
```

## Prerequisites

1. **App running:** `npm run dev` (serves on http://localhost:5000)
2. **kane-cli installed:** `npm install -g @testmuai/kane-cli@0.3.0`
3. **kane-cli authenticated:** `kane-cli login --username <user> --access-key <key>`

> **Note:** kane-cli v0.3.1–v0.3.3 have a [protobuf version mismatch bug](https://github.com/LambdaTest/kane-cli/issues/44). Use **v0.3.0** until fixed.

## Running Tests

### Via kane-cli testmd (recommended)

```bash
# Run a single test
kane-cli testmd run tests/test-cases/01_homepage_load_test.md --agent --cdp-endpoint http://localhost:9222

# Run with visible browser
kane-cli testmd run tests/test-cases/02_product_search_test.md --cdp-endpoint http://localhost:9222
```

### Via generated Playwright scripts

```bash
# Install dependencies
pip install -r tests/requirements.txt
cp tests/.env.example tests/.env  # then fill in LT_USERNAME and LT_ACCESS_KEY

# Run a script
python tests/generated-scripts/test_tc08_category.py
```

## Notes

- The `test-cases/` files are the source of truth — they define test steps in plain language
- The `generated-scripts/` are auto-generated Playwright Python code from kane-cli's code export
- The `results/` capture the pass/fail outcome of each automated run
- Partial failures are primarily viewport-related (headless Chrome not scrolling to elements below the fold)
- Test card number for checkout: `4242 4242 4242 4242`
